# 答辩提问回答提纲

原则：回答时以自己真实完成的实验为准。没做完整验证的内容不要硬说已经完成，可以回答“本项目中未作为最终主结果，但从原理上应这样处理”。

## 第一类：传感器物理与误差本质

### Q1. 六位置标定 12 参数分别对应哪些物理误差？

回答要点：

- 12 参数模型可以写成：

  `a_cal = A_inv * (a_raw - c)`

- `c` 是 3×1 零偏向量，对应 X/Y/Z 三轴在零输入时的固定偏移。
- `A_inv` 是 3×3 修正矩阵：
  - 对角线元素主要对应三轴比例因子/灵敏度误差。
  - 非对角线元素对应三轴非正交、轴间耦合、安装角误差。
- 如果只有零偏和各轴比例误差，没有轴间耦合，`A_inv` 应该退化为对角矩阵：

  `diag(kx, ky, kz)`

- 如果三轴比例也完全理想，那么 `A_inv` 接近单位矩阵，只剩下 `c` 起作用。

可答版本：

> 我的 12 参数中，3 个是零偏，9 个是比例因子和轴间耦合修正。理想情况下如果只有零偏，没有轴间耦合，A_inv 应该是对角矩阵，对角元素是各轴灵敏度修正系数；如果连比例误差也没有，它就接近单位阵。

### Q2. Allan 方差三段斜率和 KF 的 Q/R 有什么关系？

回答要点：

- 短时间段斜率约 `-1/2`：白噪声主导，对应角随机游走 ARW。
- 中段最低点：零偏稳定性最好的一段，可用于估计 bias instability。
- 长时间段斜率约 `+1/2`：零偏随机游走或慢漂移开始主导。
- 对 KF 的关系：
  - 陀螺仪白噪声影响量测噪声或角速度积分噪声，可用于设定 `R` 或陀螺测量噪声。
  - 零偏随机游走影响状态方程里的 bias 漂移，主要进入 `Q` 矩阵。

可答版本：

> Allan 曲线短时间 -1/2 斜率说明白噪声主导，也就是角随机游走；中间最低点代表零偏最稳定的时间尺度；长时间 +1/2 斜率说明慢漂移或零偏随机游走变明显。白噪声更多影响测量噪声 R，零偏随机游走对应状态噪声 Q 中 bias 状态的过程噪声。

### Q3. HMC5883L 分辨率 5 mGauss，实测 std 远大于它说明什么？

项目数据：

- 标定前磁场模长标准差：`3.303 uT`
- 标定后磁场模长标准差：`1.892 uT`
- 5 mGauss = `0.5 uT`

回答要点：

- 校准后 `1.892 uT` 仍大于数据手册分辨率 `0.5 uT`。
- 这不一定说明算法错了，因为 datasheet 分辨率只是理想量化/输出分辨率，不等于整机系统误差。
- 主要可能来自：
  - 周围铁磁物体和电流造成的磁干扰。
  - PCB 走线、电源噪声和去耦不足。
  - 采集旋转不均匀，椭球拟合残差较大。
  - HMC5883L 本身老旧，模块一致性有限。

可答版本：

> 我的标定后模长标准差约 1.892 uT，确实大于 5 mGauss，也就是 0.5 uT。这说明当前误差主要不是量化分辨率限制，而是系统级误差，包括磁环境干扰、PCB 电源和布局影响、采集覆盖不均匀以及传感器本身噪声。

### Q5. HMC5883L 离 WiFi 天线距离变化会怎样影响航向？

回答要点：

- 如果磁力计离 ESP32 WiFi 天线或大电流路径更近，可能引入电磁干扰和局部磁场扰动。
- 距离从 15 mm 减到 5 mm，干扰通常会明显增大，航向角波动和偏移都会变大。
- 实验验证方法：
  - 固定板子方向不动。
  - 分别关闭 WiFi、打开 WiFi、连续发包。
  - 记录磁场三轴、模长、yaw 标准差。
  - 比较 yaw std、模长 std 和均值偏移。

可答版本：

> 距离变近会增加磁力计受到的电磁和电源干扰，航向角可能出现固定偏移和更大波动。我会用开关 WiFi 对比实验验证：固定姿态，分别记录 WiFi 关闭和高负载发包时的磁场模长标准差与 yaw 标准差。

## 第二类：标定算法的数学深度

### Q6. 椭球拟合为什么取最小奇异值对应右奇异向量？

回答要点：

- 椭球拟合齐次方程：`Phi q = 0`。
- 不能取零解，所以约束 `||q|| = 1`。
- 目标是让残差 `||Phi q||` 最小。
- SVD 中最小奇异值对应的右奇异向量，就是单位长度条件下残差最小的方向。
- 如果数据只覆盖半个球面：
  - 某些方向约束不足。
  - 奇异值可能接近退化。
  - 解不稳定，椭球轴长和中心可能严重偏移。

可答版本：

> 因为齐次方程不能直接取 q=0，所以加单位范数约束后，本质是求使 ||Phi q|| 最小的方向。SVD 中最小奇异值对应的右奇异向量正是最小残差方向。如果只采半个球面，约束不充分，奇异值会退化，解会变得不唯一或非常不稳定。

### Q7. 桌面倾斜 2° 会进入六位置标定结果吗？如何消除？

回答要点：

- 会影响每个面的“真实重力方向”。
- 如果直接把每个面当作完全标准的 ±X/±Y/±Z，桌面倾斜会被模型吸收到 `A_inv` 的非对角项里。
- 但六位置标定本质利用的是重力模长 `|g|=1`，如果算法用模长约束或多姿态拟合，可以减小单个桌面不水平带来的影响。
- 消除方法：
  - 使用水平仪或调平平台。
  - 多次换方向采集并平均。
  - 使用更多随机姿态的球面标定，而不是只依赖六个理想面。
  - 用高精度夹具。

可答版本：

> 会进入结果。如果桌面倾斜而我还把它当成理想六方向，模型会把这部分误差吸收到安装误差或非正交矩阵里。消除方法是用水平平台、夹具，或者使用更多姿态并利用重力模长约束做自校准。

### Q8. 温度补偿二次多项式系数怎么确定？外推到 60°C 会怎样？

回答要点：

- 系数通过不同温度下静态采集零偏数据，然后最小二乘拟合 `b(T)=α+βT+γT²`。
- 20-50°C 内使用是插值。
- 60°C 使用是外推。
- 外推误差通常不可控，高阶多项式可能在区间外放大误差，甚至出现振荡。
- 属于模型外推误差/模型适用范围误差。

项目注意：

- 如果你没有完整做温度箱实验，要说“我的项目主要完成 Allan 和静态噪声分析，温度补偿属于可扩展方法”。

可答版本：

> 系数应通过不同温度点下采集陀螺零偏，然后最小二乘拟合得到。20 到 50°C 内属于插值，60°C 属于外推，误差会明显增大，因为多项式在标定区间外没有约束，甚至可能出现 Runge 现象。这属于模型外推误差。

### Q9. 离群点对 SVD 椭球拟合有什么影响？如何降低？

回答要点：

- 标准 SVD/最小二乘对离群点敏感，因为 L2 损失会放大大残差点影响。
- 一个手机靠近造成的磁异常可能拉偏椭球中心和轴长。
- 解决方法：
  - 采集前做实时模长范围检查。
  - 用 RANSAC 剔除离群点。
  - 用 Huber 损失、Tukey 损失等鲁棒估计替代普通 L2。
  - 先按模长、三轴范围、残差阈值剔除异常点，再拟合。

可答版本：

> 离群点会显著影响普通最小二乘，因为大残差会被平方放大，可能导致椭球中心和软铁矩阵被拉偏。可以用 RANSAC、Huber 损失或拟合前的模长阈值筛选来提高鲁棒性。

### Q10. BMP280 的 t_fine 是什么？温压读取相差 100 ms 会怎样？

回答要点：

- `t_fine` 是 Bosch 补偿算法中的中间温度变量。
- 它不是最终温度，而是用温度 ADC 和温度校准系数算出的高精度内部温度补偿量。
- BMP280 的压力测量受温度影响，所以压力补偿公式要使用 `t_fine`。
- 如果温度和气压读取间隔较大：
  - 在温度变化快的环境中，压力补偿使用的温度不匹配。
  - 会引入温度交叉敏感误差。
  - 100 ms 在室内慢温变下通常很小，但在热风、手触、快速气流下会变明显。

可答版本：

> t_fine 是温度补偿计算出的内部精细温度量，压力补偿也要用它，因为压力传感器本身存在温度交叉敏感。温度和压力读取相差 100 ms 时，如果环境温度变化快，就会用错误温度补偿压力，产生交叉敏感误差；室内慢温变时影响较小。

## 第三类：融合算法的工程理解

### Q11. Madgwick 的 beta 怎么选？快速机动有什么影响？

项目注意：

- 你的最终项目主要实现的是互补滤波和 Mahony 滤波，不要硬说实现了 Madgwick。

回答方式：

> 本项目最终展示的是互补滤波和 Mahony 滤波，没有把 Madgwick 作为主结果。如果从 Madgwick 原理看，beta 是梯度下降校正项权重，beta 越大，越相信加速度计/磁力计修正，收敛快但动态机动时容易被非重力加速度误导；beta 越小，越相信陀螺积分，动态响应更平滑但长期漂移更大。飞机快速急转时，加速度计测到的不再只是重力，所以 beta 应适当降低。

### Q12. Roll/Pitch 和 Yaw 精度是多少？为什么 Yaw 更差？

项目数据：

- ESKF roll std：`0.074643°`
- ESKF pitch std：`0.132308°`
- 静态稳定 RMS：`0.151800°`
- 航向残差补偿后平均绝对误差：`1.907°`
- 航向残差补偿后最大绝对误差：`6.940°`

回答要点：

- Roll/Pitch 可由加速度计观测重力方向，因此静态可观测。
- Yaw 是绕重力方向旋转，加速度计无法约束，必须依赖磁力计或 GNSS/视觉等外部参考。
- 去掉磁力计后，Yaw 只能靠陀螺积分，会随零偏漂移。
- 10 分钟漂移估计：`漂移角 ≈ 陀螺零偏 deg/s × 600 s`。

可答版本：

> 静态实验中 Roll/Pitch 的 1σ 稳定性约 0.075° 和 0.132°，RMS 约 0.152°。Yaw 经过残差补偿后平均绝对误差约 1.91°，最大约 6.94°。Yaw 更差是因为加速度计只能约束重力方向，也就是 Roll/Pitch，绕重力轴的 Yaw 对加速度计不可观，必须依赖磁力计，而磁力计又容易受环境干扰。若去掉磁力计，10 分钟漂移可按陀螺零偏乘 600 秒估算。

### Q13. KF 的 Q 矩阵代表什么？Q 过大过小会怎样？

回答要点：

- `Q` 是过程噪声协方差，表示模型预测本身有多不可靠。
- `R` 是观测噪声协方差，表示传感器观测有多不可靠。
- Q 太小：
  - 滤波器过度相信模型。
  - 对真实变化反应慢。
  - 可能出现过度自信，严重时发散。
- Q 太大：
  - 过度相信观测。
  - 曲线变抖，滤波效果变差。
  - 近似退化为直接跟随测量。

项目注意：

- 如果问高度 KF，你可以说本项目主要完成 BMP 相对高度实验和静态分析，KF 参数更多是工程扩展。

可答版本：

> Q 表示状态模型的不确定性。Q 设得太小，滤波器会过度相信预测，真实高度变化时跟踪慢，严重时出现 over-confidence 甚至发散；Q 设得太大，则过度相信观测，输出会抖，滤波效果减弱。

### Q14. GPS/气压互补 alpha=0.98 对应时间常数？台风气压漂移怎么办？

项目注意：

- 你没有把 GPS/气压动态互补作为最终主结果，回答时先说明。

原理回答：

- 一阶低通/互补滤波时间常数近似：`tau = dt / (1 - alpha)`。
- 如果 `dt = 0.02 s`，`alpha=0.98`，则 `tau ≈ 1 s`。
- 气压变化 1 hPa 约对应 8.4 m 高度变化。
- 3 hPa 约对应 `25.2 m` 高度误差。
- 气象慢漂会完全渗透到气压高度中，因此需要：
  - 定期用 GPS 或已知高度重置 `P0`。
  - 引入气象站参考气压。
  - 对气压慢漂单独建模。
  - 动态调整 alpha。

可答版本：

> 本项目没有把 GPS/气压动态互补作为最终主结果。如果按 alpha=0.98、dt=0.02 s 估算，时间常数约 1 s。台风前 1 小时下降 3 hPa，会造成约 25 m 的气压高度漂移，因此必须定期用 GPS 或已知高度重置 P0，不能长期只信气压高度。

### Q15. I2C 多设备读取怎么调度？中断竞争怎么处理？

回答要点：

- I2C 是主从总线，ESP32-S3 是主设备，所有读写由主设备发起。
- 同一时刻总线上只能有一个事务。
- MPU 中断只是提醒数据就绪，不会物理上打断正在进行的 BMP280 I2C 读操作。
- 程序上应该：
  - 高频读取 IMU。
  - 磁力计、BMP280 低频读取。
  - 所有 I2C 访问用同一任务串行调度，或用 mutex 保护。
  - 中断中只置标志位，不直接做 I2C 读写。

可答版本：

> I2C 是主设备控制时序，所以即使 MPU 中断在 BMP280 读取中触发，也不会打断当前 I2C 事务。正确做法是中断里只置 data-ready 标志，在任务里按优先级调度读取；如果多任务访问 I2C，则必须用 mutex 保证一次只有一个 I2C 事务。

## 第四类：系统工程与批判性思维

### Q16. 安装倾斜能被六位置标定补偿吗？

回答要点：

- 如果传感器固定在 PCB 上，且标定后不再移动，安装倾斜可以被 12 参数模型中的非对角项吸收。
- 补偿发生在 `A_inv * (raw - c)` 这一步。
- 如果传感器不是固定安装，比如杜邦线每次弯折位置不同，那么安装关系变化，旧标定参数失效。

可答版本：

> 可以补偿固定安装后的安装角误差，主要体现在 A_inv 的非对角项里。前提是标定后传感器和板子的相对姿态不再变化。如果用杜邦线悬空连接，每次姿态都变，就不能靠一次标定长期补偿。

### Q17. 无人机电机启动后磁力计跳变怎么办？

方案：

1. 磁力计远离电机和电源线，例如延伸杆安装。
   - 优点：从源头降低干扰。
   - 缺点：结构复杂，线缆更长。
2. 做电流补偿，类似 ArduPilot COMPASS_MOT。
   - 优点：能补偿随电流变化的磁干扰。
   - 缺点：需要电流传感器和标定。
3. 飞行中降低磁力计权重，更多依赖陀螺、GPS 航向或视觉。
   - 优点：动态抗干扰更好。
   - 缺点：低速或悬停时 yaw 约束变弱。
4. 双天线 GNSS 测向替代磁罗盘。
   - 优点：不受电机磁场影响。
   - 缺点：成本和尺寸更高。

可答版本：

> 我会优先把磁力计远离电机和大电流线，同时做电流相关补偿。如果是大型平台，可以考虑双天线 GNSS 测向。单纯屏蔽不一定有效，因为低频磁场很难靠普通屏蔽解决。

### Q18. 如果重新设计 PCB，改哪些 Layout？

可答三点：

1. 磁力计远离 ESP32 天线、USB、电源和大电流走线。
   - 预期收益：降低磁场偏置和 yaw 波动。
2. 传感器电源去耦电容更靠近 VCC/GND 引脚。
   - 预期收益：降低电源纹波和 I2C/ADC 噪声，改善 uT 级磁场波动。
3. I2C 走线短、等长、有连续参考地，避免跨分割地。
   - 预期收益：降低通信毛刺，提高 400 kHz I2C 稳定性。
4. 地平面连续铺铜，模拟传感器下方避免高速数字线穿过。
   - 预期收益：降低 EMI 耦合。

如果不知道铺铜覆盖率，不要乱报：

> 我没有在答辩前精确计算背面铺铜覆盖率，但从工程角度，下一版会优先保证连续地平面和磁力计区域远离高频/大电流干扰源。

回答要点：

- Roll/Pitch 瓶颈主要是加速度计标定残差和静态平台误差。
- Yaw/航向瓶颈主要是磁环境，不是算法本身。
- 高度瓶颈主要是 P0 和环境气压漂移，不是 BMP280 分辨率。
- 纯软件还能提升：
  - 航向通过鲁棒椭球拟合、在线磁异常检测可能提升。
  - 高度通过 GPS/气压/已知高度重置 P0 改善长期漂移。
  - Roll/Pitch 提升空间较小，更多依赖机械固定和标定条件。

### Q20. 装到真实大型垂起固定翼还缺什么？

优先改进三点：

1. 航向系统可靠性
   - 磁力计延伸杆、电流补偿、双 GNSS 测向。
   - 预期：降低电机磁干扰，航向误差从数度级向 2°以内逼近。
2. 动态 GNSS/INS 融合
   - 真正实现 IMU 高频预测 + GNSS 低频校正。
   - 增加丢星、延迟、异常值检测。
   - 预期：提升动态位置和速度估计稳定性。
3. 工程可靠性
   - 电源冗余、EMC、振动隔离、温度补偿、日志系统。
   - 预期：提升飞行安全性，而不仅是静态精度。

可答版本：

> 如果装到真实 VTOL，无人机电机、电源和振动都会比实验室复杂得多。我会优先改航向可靠性、动态 GNSS/INS 融合和工程可靠性。特别是磁力计必须远离电机并做电流补偿，动态融合要考虑 GPS 延迟、丢星和异常点，硬件上还要考虑振动、电源冗余和 EMI。
