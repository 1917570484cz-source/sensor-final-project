# 二、系统设计要求

## 1. 硬件设计要求（ESP32 扩展板）

### 1.1 基础规格

- 尺寸限制：≤70mm×50mm（适配嘉立创打样尺寸，标准 2 层板）
- 供电：通过 ESP32 开发板 5V/3.3V 供电，无需外部电源
- 接口：
  - 全部采用 2.54mm 排针，与 ESP32 开发板引脚一一对应
  - I²C 总线共享（SCL/SDA），所有传感器挂同一 I²C 总线
  - GPS 接口：4-pin 2.54mm 排针（VCC/GND/TX/RX），外接 NEO-6M，用杜邦线连
  - LED 指示：至少 1 个电源指示 LED，绿色，串 1kΩ 电阻
- 布局建议：
  - MPU6050 放板中央，减少边缘应力，距板边 ≥ 5mm
  - HMC5883L 远离 ESP32，避免 ESP32 芯片 / WiFi 天线的磁干扰
  - 远离发热元件，避免温度影响气压读数
  - GPS 排针在板边缘，方便连线朝外
  - I²C 上拉电阻一组就够（多个会并联降阻），4.7kΩ × 2（SCL/SDA 各一个）

### 1.2 必修传感器电路

#### MPU6050 六轴 IMU

- 通信接口：SPI（优先）或 I2C
- 必须包含 0.1μF 去耦电容、电源滤波电路
- 预留中断引脚，用于数据就绪触发

#### HMC5883L 三轴磁力计

- 通信接口：I2C
- 必须包含去耦电容和上拉电阻
- 布局远离电源、电机等强干扰源

### 1.3 选修传感器电路（鼓励至少选 1 种）

- MS5611 或 BMP280 气压计：SPI 接口，高精度气压高度测量
- Ublox M6N GPS 模块：UART 接口，预留天线接口
- 电源管理：可选 3.3V LDO，为传感器提供干净电源

### 1.4 信号完整性要求（对应第六讲内容）

- 所有传感器电源引脚必须就近放置 0.1μF 陶瓷去耦电容
- 模拟信号走线尽量短，避免与数字信号走线交叉
- 完整的地平面设计，分割数字地和模拟地
- 预留测试点，方便调试和信号测量

## 2. 算法设计要求

### 2.1 基础算法（所有组必须完成）

#### 传感器标定算法

- 加速度计六位置标定（12 参数模型），零偏 50mg → 2mg（25×）
- 磁力计椭球拟合标定（硬铁 / 软铁补偿），航向 ±15° → ±2°（7×）
- 陀螺仪零偏校准与 Allan 方差分析，零偏漂移降 10×

#### 传统姿态融合算法（至少实现 2 种）

- 互补滤波
- Mahony 滤波
- Madgwick 滤波
- 扩展卡尔曼滤波（EKF）

#### 性能对比分析

- 对比不同算法的姿态精度、计算复杂度、CPU 占用率
- 量化每种算法带来的精度提升幅度

#### 数据可视化（必做）

| 内容 | 工具 / 方式 | 要求 |
| --- | --- | --- |
| 标定前后精度对比表 | Python matplotlib | 误差降低倍数 |
| Allan 方差曲线 | Python allantools | ARW/BI 标注 |
| 磁力计 3D 散点（校准前后） | Python mplot3d | 椭球 → 球 |
| 实时姿态显示 | 串口 + Python / Web | 倾斜板子看角度变化 |
| GPS 轨迹叠加 | Python folium | 户外测试轨迹 |

### 2.2 进阶算法（80 分以上必须完成）

#### 误差状态卡尔曼滤波（ESKF）

- 基于李群李代数 SO(3) 的姿态表示
- 实现 IMU 预积分与误差状态更新
- 目标姿态精度：≤±0.2°（静态）

#### AI 增强传感器算法（至少实现 1 种）

- 1D-CNN IMU 噪声去除：对比传统低通滤波的 SNR 提升 dB 数
- 一阶卡尔曼滤波与 1D-CNN 的混合去噪架构
- 目标：纯惯导漂移从 10m/min 降低至 ≤3m/min

### 2.3 挑战算法（90 分以上可选）

- 多速率传感器融合：GPS 1Hz + IMU 200Hz 异步状态更新
- LSTM GPS 失锁预测：当 GPS 信号丢失时，用 LSTM 预测轨迹
- TLIO/DeepVIO 核心模块复现：基于深度学习的紧耦合惯导里程计
- 多传感器时间同步：实现 IMU、磁力计、GPS 的微秒级时间对齐
