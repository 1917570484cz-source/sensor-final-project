# 第一类：传感器物理与误差本质

## Q1. 六位置标定得到 12 个参数，它们对应 MPU6050 的哪些物理误差？

### 1. 老师到底在问什么

老师不是只想听你说“我用了最小二乘”。他想确认你知道标定出来的数字到底修正了什么物理问题。

你加速度计的标定模型可以写成：

```text
a_cal = A_inv · (a_raw - c)
```

含义是：

```text
校正后的加速度 = 3×3 修正矩阵 × (原始加速度 - 零偏)
```

其中：

- `a_raw`：MPU6050 原始读数。
- `c`：3×1 零偏向量。
- `A_inv`：3×3 误差修正矩阵。
- `a_cal`：校正后的加速度。

所以 12 个参数就是：

```text
3 个零偏参数 + 9 个矩阵参数 = 12 个参数
```

### 2. c 向量对应什么误差

`c` 是：

```text
c = [cx, cy, cz]^T
```

它对应三轴零偏误差。

比如板子水平静止时，理论上应该是：

```text
ax = 0
ay = 0
az = 1g
```

但实际可能读到：

```text
ax = 0.02g
ay = -0.01g
az = 1.05g
```

其中 `ax`、`ay` 明明应该接近 0，却读出非零值，这就是零偏。`az` 理论上应该接近 1g，却多出一部分，也包含零偏和比例误差。

所以第一步 `(a_raw - c)` 就是先把固定偏移减掉。

### 3. A_inv 矩阵对应什么误差

`A_inv` 是一个 3×3 矩阵：

```text
A_inv =
[ a11  a12  a13
  a21  a22  a23
  a31  a32  a33 ]
```

它主要修正三类误差。

第一类是比例因子误差，也叫灵敏度误差。

比如 X 轴真实受到 1g 加速度，理想输出应该是：

```text
ax = 1.000g
```

但实际可能输出：

```text
ax = 0.980g
```

或者：

```text
ax = 1.030g
```

说明 X 轴这把“尺子”不准。`A_inv` 的对角线元素主要修正三轴比例误差：

```text
a11 修正 X 轴比例
a22 修正 Y 轴比例
a33 修正 Z 轴比例
```

第二类是轴间耦合误差。

理想情况下，X 轴只测 X 方向，Y 轴只测 Y 方向，Z 轴只测 Z 方向。但实际传感器三轴不可能完全独立。

比如真实加速度只有 X 方向：

```text
真实加速度 = [1g, 0, 0]
```

理想读数应该是：

```text
输出 = [1g, 0, 0]
```

但实际可能是：

```text
输出 = [1g, 0.03g, -0.02g]
```

这说明 X 方向的加速度串到了 Y/Z 轴上。`A_inv` 的非对角线元素就是用来修正这种耦合：

```text
a12, a13, a21, a23, a31, a32
```

第三类是安装角误差和三轴不正交误差。

即使 MPU 芯片本身正常，它焊在 PCB 上也可能有非常小的倾斜。比如你以为板子的 X 方向就是传感器 X 方向，但芯片实际歪了 1°。这样真实 X 方向的加速度会投影到 Y/Z 轴，表现出来也像轴间耦合。这部分同样由 `A_inv` 的非对角线修正。

### 4. 如果只有零偏误差，A_inv 应该长什么样

要分两种情况说，最完整。

第一种：只有零偏，比例因子和三轴方向都完全理想。

这时只需要减掉 `c`，不需要矩阵修正，所以：

```text
A_inv =
[1  0  0
 0  1  0
 0  0  1]
```

也就是单位矩阵。

第二种：有零偏，也有各轴比例误差，但没有轴间耦合。

这时 `A_inv` 应该是对角矩阵：

```text
A_inv =
[kx  0   0
 0   ky  0
 0   0   kz]
```

其中 `kx`、`ky`、`kz` 分别修正 X/Y/Z 三轴的灵敏度。非对角线为 0，表示 X 不影响 Y/Z，Y 不影响 X/Z，Z 不影响 X/Y。

### 5. 结合本项目怎么说

本项目中，六位置 12 参数标定后：

- 平均误差从约 `20.068 mg` 降到 `1.798 mg`。
- 最大误差从约 `55.789 mg` 降到 `2.120 mg`。

这说明零偏、比例误差和安装耦合误差都被明显补偿了。

### 6. 可直接背的回答

> 我的加速度计标定模型是 `a_cal = A_inv · (a_raw - c)`。其中 `c` 是 3×1 零偏向量，表示 X、Y、Z 三轴的固定偏移。`A_inv` 是 3×3 修正矩阵，对角线元素主要修正三轴灵敏度或比例因子误差，非对角线元素主要修正轴间耦合、三轴不正交以及传感器安装角误差。如果只有零偏且比例完全理想，`A_inv` 应该接近单位矩阵；如果还有比例误差但没有轴间耦合，`A_inv` 应该退化为对角矩阵，对角元素是各轴灵敏度修正系数，非对角线接近 0。

### 7. 可能追问

如果老师问“为什么叫 A_inv？”：

> 可以把原始传感器看成真实加速度先经过一个误差矩阵 A 扭曲后输出。标定的目的就是求它的逆，把扭曲后的数据拉回真实坐标系，所以代码中叫 `A_inv`。

如果老师问“c 和 A_inv 谁先作用？”：

> 先减零偏 `c`，再乘 `A_inv`。因为零偏是固定偏移，先去掉固定偏移，再修正比例和轴间耦合。

## Q2. Allan 方差曲线短时间 -1/2、中段最低点、长时间 +1/2 分别代表什么？

### 1. 老师到底在问什么

老师想知道你是否理解 Allan 方差不是普通曲线，而是用来分析 IMU 噪声来源的工具。

Allan 方差横轴通常是平均时间 `τ`，纵轴是 Allan deviation。它回答的问题是：

> 如果我把陀螺仪数据按不同时间窗口平均，噪声会怎么变化？

不同斜率对应不同噪声源。

### 2. 短时间段斜率 -1/2 是什么

短时间段通常由白噪声主导。对白噪声来说，平均时间越长，随机噪声越容易被平均掉，所以 Allan deviation 会下降。

在双对数图上，白噪声对应斜率约为：

```text
-1/2
```

对陀螺仪来说，这段常用来估计角随机游走 ARW，即 Angular Random Walk。

通俗理解：

> 短时间里，陀螺仪输出主要是在真实值附近随机抖动，这种抖动可以通过平均减小。

### 3. 中段最低点是什么

曲线下降到某个位置后会出现最低点或平台，这附近表示平均到这个时间尺度时噪声最小。

这一段通常和零偏不稳定性 BI，也就是 Bias Instability 有关。

通俗理解：

> 平均时间太短，白噪声还没平均干净；平均时间太长，零偏慢漂移又开始显现。中间最低点就是两类误差折中后最稳定的位置。

### 4. 长时间段斜率 +1/2 是什么

长时间段如果曲线上升，常表示零偏随机游走、温漂、慢变化环境等低频漂移开始主导。

在双对数图上，如果斜率约为：

```text
+1/2
```

通常可理解为 rate random walk 或 bias random walk 类型的慢漂移。

通俗理解：

> 时间越长，陀螺仪零点不是固定的，而是慢慢飘走，所以平均反而不能继续降低误差。

### 5. 和 Kalman 滤波 Q、R 的关系

Kalman 滤波里：

- `R` 表示观测噪声，也就是传感器测量本身有多吵。
- `Q` 表示过程噪声，也就是系统状态预测有多不可靠。

对应关系可以这样理解：

短时间白噪声/角随机游走：

- 反映陀螺仪测量噪声。
- 可以用于设置陀螺仪测量噪声或姿态更新里的噪声项。
- 更接近 `R` 或角速度测量噪声。

长期零偏漂移：

- 反映陀螺仪 bias 会随时间变化。
- 如果滤波器把陀螺零偏作为状态估计，那么 bias 的随机游走噪声应该放进 `Q`。

### 6. 结合本项目怎么说

本项目完成了约 300 s 静态陀螺仪数据采集和 Allan 方差曲线绘制。它主要用于分析噪声特性和辅助滤波器参数选择。

注意不要说“已经严格证明零偏漂移降低 10×”，因为这个需要补偿前后长期对比数据，而当前主要是 Allan 噪声分析。

### 7. 可直接背的回答

> Allan 方差短时间段斜率约 -1/2，说明白噪声主导，对陀螺仪来说对应角随机游走 ARW；中间最低点表示在这个平均时间尺度下零偏最稳定，和 bias instability 有关；长时间段如果斜率变成 +1/2，说明零偏随机游走或慢漂移开始主导。和 Kalman 滤波对应时，短时间白噪声更多用于设置陀螺测量噪声，也就是 R 或角速度噪声；长期零偏漂移则对应状态模型里 bias 的过程噪声，应该进入 Q 矩阵。

### 8. 可能追问

如果老师问“为什么不是平均越久越准？”：

> 因为平均只能降低白噪声，但不能消除零偏慢漂移。时间太长后，零偏本身会变，平均反而把漂移包含进去，所以 Allan 曲线会重新上升。

## Q3. HMC5883L 分辨率是 5 mGauss，为什么校准后磁场模长 std 仍然远大于它？

### 1. 老师到底在问什么

老师想看你是否能区分：

- 数据手册里的分辨率
- 实际系统中的精度
- 实验环境带来的干扰

很多人会误以为传感器分辨率就是最终精度，这是错误的。

### 2. 单位先换算

HMC5883L 分辨率是 5 mGauss。

换算关系：

```text
1 Gauss = 100 uT
1 mGauss = 0.1 uT
5 mGauss = 0.5 uT
```

所以数据手册分辨率约为：

```text
0.5 uT
```

### 3. 本项目实测数据

本项目磁力计模长结果：

- 标定前模长标准差：`3.303 uT`
- 标定后模长标准差：`1.892 uT`

校准后 `1.892 uT` 仍然大于数据手册分辨率 `0.5 uT`。

### 4. 为什么会这样

原因是 datasheet 分辨率只说明传感器输出能分辨的最小变化，不等于最终整机精度。

实际系统中还有很多误差：

1. 周围磁环境干扰
   - 桌面、螺丝、手机、电源线、电脑、USB 线都可能影响磁场。

2. PCB 布局和电源噪声
   - 磁力计附近如果有高频数字信号、大电流走线或去耦不好，会增加噪声。

3. 硬铁和软铁残差
   - 椭球拟合能补偿一部分，但如果采集数据覆盖不均匀，补偿不可能完全理想。

4. 采集动作不均匀
   - 如果没有完整覆盖整个球面，某些方向的数据不足，拟合结果会偏。

5. 传感器本身老旧或模块差异
   - HMC5883L 属于较老的磁力计，实际模块质量差异较大。

### 5. 算法不对还是硬件原因

不能简单说算法不对。

如果标定后标准差从 `3.303 uT` 降到 `1.892 uT`，说明算法确实有效。但是剩余误差大于 datasheet 分辨率，主要说明系统级环境和硬件因素仍然存在。

### 6. 可直接背的回答

> HMC5883L 的 5 mGauss 约等于 0.5 uT。我的磁力计标定前模长标准差是 3.303 uT，标定后是 1.892 uT，虽然明显下降，但仍然大于 0.5 uT。这不一定说明算法错误，因为 datasheet 分辨率不等于整机精度。剩余误差主要来自磁环境干扰、PCB 电源和走线影响、采集球面覆盖不均匀，以及硬铁软铁补偿残差。

### 7. 可能追问

如果老师问“怎么进一步降低？”：

> 我会让磁力计远离 ESP32 天线、USB 和电源走线，增加去耦，采集时远离手机和铁磁物体，并用更均匀的全空间旋转数据进行鲁棒椭球拟合。

## Q4. 气压计测高精度是多少？误差预算怎么分解？

### 1. 老师到底在问什么

老师想知道你是否明白：BMP280 测高度不是直接测“高度”，而是先测气压，再用气压换算高度。

所以误差不仅来自 BMP280 本身，还来自参考气压、温度、天气变化和真实高度参考。

### 2. 本项目结果

楼梯实验数据：

- 人工测量楼层高度：`3.950 m`
- BMP280 测量相对高度：`3.980 m`
- 绝对误差：`0.030 m`
- 相对误差：`0.76%`

也就是说，在这次短时间楼梯相对高度实验中，误差约为 3 cm。

### 3. 气压高度为什么依赖 P0

气压高度不是直接测出来的，而是由气压公式换算：

```text
h = f(P / P0)
```

其中：

- `P` 是当前气压。
- `P0` 是参考气压。

如果 `P0` 不准，即使 BMP280 当前压力测得很准，高度也会整体偏。

### 4. 误差预算怎么拆

可以拆成以下几项：

1. 传感器压力噪声
   - BMP280 自身压力读数会有噪声。
   - 短时间平均可以降低这部分噪声。

2. P0 参考气压误差
   - 高度换算对 P0 很敏感。
   - P0 不准会造成整体高度偏移。
   - 在绝对高度测量中，这通常是主导误差。

3. 温度补偿误差
   - BMP280 压力补偿依赖温度。
   - 温度变化或温度读取滞后会影响压力换算。

4. 天气漂移
   - 气压会随天气变化。
   - 长时间测试时，天气变化可能造成几十米级高度漂移。
   - 但本项目楼梯实验时间短，所以影响较小。

5. 人工真值误差
   - 楼层高度 3.95 m 是人工测量值，也有测量误差。

6. 气流和动态扰动
   - 上下楼、开门、风流会改变局部气压。

### 5. 哪一项是主导误差

在本项目的短时间楼梯相对高度实验中：

- 主要瓶颈不是 BMP280 分辨率。
- 更主要的是 `P0` 参考、环境气压稳定性、人工测量真值和窗口平均方法。

在长时间绝对高度实验中：

- `P0` 和天气漂移通常是主导误差。

### 6. 可直接背的回答

> 我这次 BMP280 楼梯实验中，人工测量高度是 3.950 m，BMP280 测得 3.980 m，绝对误差 0.030 m，相对误差 0.76%。误差预算可以分为传感器压力噪声、P0 参考气压误差、温度交叉敏感、天气气压漂移、人工高度真值误差和气流扰动。短时间相对高度实验中，BMP280 分辨率不是主要瓶颈，主要瓶颈是 P0 和环境条件；长时间绝对高度中，天气漂移和 P0 会成为主导误差。

## Q5. HMC5883L 离 ESP32 WiFi 天线距离变化会怎样影响航向？

### 1. 老师到底在问什么

老师想看你是否从系统电磁兼容角度理解磁力计，而不是只看算法。

磁力计测的是微弱地磁场，地磁场量级大约几十 uT。任何附近电流、天线、高频数字信号、铁磁材料都可能改变磁场读数。

### 2. 距离从 15 mm 变 5 mm 会怎样

如果磁力计离 ESP32 WiFi 天线或电源路径更近，可能会出现：

- 磁场三轴均值偏移。
- 磁场模长标准差变大。
- 航向 yaw 出现固定偏差。
- WiFi 发射时 yaw 抖动增加。

距离变近通常会让干扰增强。具体增强多少取决于天线、电流路径、地平面和磁力计方向。

### 3. 怎么实验验证

可以设计一个很简单的对比实验：

1. 固定板子，不让它动。
2. 记录 WiFi 关闭时的磁场三轴和 yaw。
3. 打开 WiFi 但不传输数据，再记录。
4. WiFi 高负载发包，再记录。
5. 比较三种情况下：
   - `mag_x/y/z` 均值变化。
   - 磁场模长标准差。
   - yaw 平均值偏移。
   - yaw 标准差。

如果 WiFi 影响明显，高负载发包时 yaw 标准差和模长标准差会变大。

### 4. 本项目怎么说

你没有精确测 HMC5883L 到 WiFi 天线距离，也没有做开关 WiFi 对比实验，所以不要乱报具体数值。

可以回答：

> 这一项我没有单独量化，但从 EMC 角度，下一版 PCB 会让磁力计远离 ESP32 天线、电源和 USB 区域。

### 5. 可直接背的回答

> 磁力计测的是几十 uT 量级的地磁场，离 ESP32 WiFi 天线或大电流路径太近，会引入电磁干扰和局部磁场扰动。距离从 15 mm 减到 5 mm，我预期磁场模长波动和 yaw 标准差都会增大。验证方法是固定板子，分别在 WiFi 关闭、开启、持续发包三种情况下采集磁场和 yaw，比较均值偏移和标准差变化。

# 第二类：标定算法的数学深度

## Q6. 椭球拟合 SVD 为什么取最小奇异值对应的右奇异向量？

### 1. 老师到底在问什么

老师想确认你知道 SVD 在这里不是“固定套路”，而是在求一个最小残差解。

椭球拟合可以写成齐次方程：

```text
Phi · q = 0
```

其中：

- `Phi` 是由磁力计采样点构成的矩阵。
- `q` 是椭球方程参数。

### 2. 为什么不能直接解

因为这是齐次方程，`q = 0` 永远满足：

```text
Phi · 0 = 0
```

但 `q=0` 没有任何物理意义。所以要加一个约束：

```text
||q|| = 1
```

也就是在单位长度的所有 `q` 里面，找一个让残差最小的方向。

目标变成：

```text
min ||Phi · q||
subject to ||q|| = 1
```

### 3. SVD 为什么取最小奇异值

SVD 分解：

```text
Phi = U · S · V^T
```

奇异值表示矩阵在不同方向上的放大能力。

最小奇异值对应的右奇异向量，就是被 `Phi` 放大最小的方向。也就是说，在单位长度条件下，它让：

```text
||Phi · q||
```

最小。

所以取最小奇异值对应的右奇异向量，而不是最大奇异值。最大奇异值方向是残差最大的方向，正好相反。

### 4. 如果数据只覆盖半个球面会怎样

椭球拟合要求数据尽量覆盖完整三维空间。如果只转了半个球面，会出现：

- 某些方向没有约束。
- 奇异值分布退化。
- 最小奇异值对应的方向不稳定。
- 椭球中心和轴长可能严重偏。
- 解可能不唯一。

通俗说：

> 数据没有告诉算法“另一半球面长什么样”，算法只能猜，所以拟合会不可靠。

### 5. 可直接背的回答

> 椭球拟合可以写成齐次方程 `Phi q = 0`。因为零解没有意义，所以加 `||q||=1` 的约束，本质上是在单位长度的 q 中找让 `||Phi q||` 最小的方向。SVD 中最小奇异值对应的右奇异向量正是最小残差方向，所以取它。如果采集数据只覆盖半个球面，某些方向约束不足，奇异值会退化，解会变得不唯一或不稳定，椭球中心和轴长都可能偏。

## Q7. 六位置标定时桌面不水平，误差会进入标定结果吗？

### 1. 老师到底在问什么

老师想看你是否理解“六个面的真值”不是天然准确的。你把板子放在桌面上，默认桌面是水平的。但如果桌面倾斜 2°，理论真值就不是完美的 ±X/±Y/±Z。

### 2. 桌面倾斜会造成什么

假设你以为当前是 +Z 朝上，理论加速度应为：

```text
[0, 0, 1g]
```

但桌面倾斜 2° 时，重力会在 X/Y 方向有投影。

小角度下：

```text
sin(2°) ≈ 0.0349
```

也就是会产生约：

```text
0.035g = 35 mg
```

的横向分量。

这已经比 2 mg 目标大很多。

### 3. 误差会不会进入标定结果

会。

如果你仍然把这个位置当作理想 +Z，那么标定算法会认为这个横向分量是传感器误差，并把它吸收到：

- 零偏 `c`
- 非正交矩阵 `A_inv`
- 安装角误差项

里。

这样标定出来的参数会包含桌面倾斜误差。

### 4. 如何消除

方法：

1. 使用水平仪或高精度水平平台。
2. 每个方向多次摆放并取平均。
3. 用夹具保证六个方向正交。
4. 不只用六位置，而是采集更多随机姿态，用重力模长 `|g|=1` 约束做球面标定。
5. 标定时记录平台倾角并修正真值方向。

### 5. 本项目怎么说

你的项目中确实是手动六位置采集，所以可以承认：

> 摆放误差会影响标定结果，因此我通过每个位置采集多样本并筛选稳定数据来降低随机影响，但如果平台系统性倾斜，仍然会进入标定参数。

### 6. 可直接背的回答

> 会进入。因为六位置标定默认六个面的真值是 ±X、±Y、±Z，如果桌面倾斜 2°，重力会产生约 sin2°≈0.035g，也就是 35mg 的横向分量。算法可能把这个误差吸收到非正交矩阵或安装角误差里。消除方法是使用水平平台或夹具，或者采集更多随机姿态，用重力模长约束做自校准。

## Q8. 温度补偿二次多项式系数怎么确定？外推到 60°C 会怎样？

### 1. 老师到底在问什么

老师想确认你知道标定模型有适用范围。一个模型在标定温度范围内可能有效，但超出范围就不一定可靠。

### 2. 二次温度补偿模型是什么

陀螺仪零偏会随温度变化，可以建模为：

```text
b(T) = α + βT + γT²
```

其中：

- `b(T)` 是某一温度下的零偏。
- `T` 是温度。
- `α、β、γ` 是拟合系数。

三轴陀螺通常每个轴都要拟合一组三个系数。

### 3. 系数怎么确定

需要在多个温度点下采集静态陀螺数据：

```text
T1 -> b1
T2 -> b2
T3 -> b3
...
```

然后用最小二乘拟合：

```text
b(T) = α + βT + γT²
```

温度点越多、覆盖越均匀，拟合越可靠。

### 4. 20°C 到 50°C 和 60°C 的区别

如果标定数据覆盖 20°C 到 50°C：

- 在 20-50°C 内使用，叫插值。
- 在 60°C 使用，叫外推。

插值相对可靠，因为模型在已知数据范围内。

外推不可靠，因为 60°C 没有数据约束，二次多项式可能偏离真实传感器特性。

### 5. 外推误差属于什么误差

属于模型外推误差，也可以说是模型适用范围外误差。

如果多项式阶数更高，还可能出现 Runge 现象，即在区间外或边缘出现剧烈振荡。

### 6. 项目边界

如果老师问“你的温度补偿”，但你没有完整做温箱实验，不要硬说已经做完。

你可以说：

> 本项目主要完成 Allan 方差和静态噪声分析，温度补偿作为可扩展方法；如果要做，需要按温度点采集零偏并拟合系数。

### 7. 可直接背的回答

> 二次温度补偿系数需要在多个温度点下静态采集陀螺零偏，然后用最小二乘拟合 `b(T)=α+βT+γT²`。20°C 到 50°C 内使用属于插值，60°C 使用属于外推。外推没有实测数据约束，误差可能明显增大，属于模型外推误差；高阶多项式还可能在区间外产生振荡。

## Q9. 磁力计椭球拟合中离群点有什么影响？如何减少？

### 1. 老师到底在问什么

老师想看你是否知道普通最小二乘对异常点很敏感。

### 2. 为什么离群点影响大

SVD 椭球拟合本质上是最小二乘思想，最小化所有点的整体残差。

普通最小二乘相当于使用 L2 损失：

```text
loss = residual²
```

残差会被平方，所以一个很大的离群点会被赋予非常大的影响。

比如采集时手机靠近一下，磁力计读数突然跳变，这个点可能把椭球中心和轴长拉偏。

### 3. 离群点会造成什么结果

可能导致：

- 硬铁中心估计错误。
- 软铁矩阵估计错误。
- 校准后球面仍然不圆。
- 航向角出现固定偏差或局部方向误差。

### 4. 如何减少离群点影响

方法：

1. 采集时实时看磁场模长，明显异常就重采。
2. 标定前按模长范围筛选数据。
3. 用 RANSAC：
   - 随机选一部分点拟合。
   - 看有多少点符合模型。
   - 保留内点最多的模型。
4. 用 Huber 损失或 Tukey 损失替代普通 L2。
5. 拟合后根据残差再次剔除异常点并重新拟合。

### 5. 可直接背的回答

> 离群点对普通 SVD 椭球拟合影响很大，因为它本质上接近最小二乘，L2 损失会把大残差平方放大。一个手机靠近造成的异常点可能拉偏椭球中心和轴长。减少影响的方法包括采集前后做模长阈值筛选、用 RANSAC 找内点、用 Huber 等鲁棒损失，或者先拟合再按残差剔除异常点重新拟合。

## Q10. BMP280 的 t_fine 是什么？为什么温度补偿结果会用于气压补偿？

### 1. 老师到底在问什么

老师想确认你不是直接复制 Bosch 代码，而是理解温度和压力补偿之间的耦合。

### 2. t_fine 是什么

BMP280 里温度 ADC 不是直接等于真实温度，压力 ADC 也不是直接等于真实压力。

Bosch 补偿算法先根据温度原始值和芯片内部校准参数算出一个中间量：

```text
t_fine
```

它可以理解为高精度内部温度补偿状态。

它不是简单的摄氏温度显示值，而是用于后续补偿计算的内部变量。

### 3. 为什么气压补偿也要用 t_fine

压力传感器对温度敏感。也就是说，同一个真实压力下，如果温度不同，压力 ADC 输出也会变。

所以压力补偿时必须知道当前温度状态。

Bosch 的算法就是：

1. 先用温度原始值计算 `t_fine`。
2. 再用 `t_fine` 修正压力原始值。

这说明温度补偿和压力补偿不是完全独立的。

### 4. 温度和气压读取时间差 100 ms 会怎样

如果温度和气压读取差了 100 ms，在室内慢温变条件下影响可能很小。

但如果环境温度变化很快，比如：

- 手指触摸芯片
- 热风吹过
- 设备从室内到室外
- 电路发热快速变化

那么压力补偿使用的温度就不是压力采样时的真实温度，会引入温度交叉敏感误差。

### 5. 可直接背的回答

> t_fine 是 Bosch 补偿算法中的内部精细温度变量，由温度原始 ADC 和芯片校准参数计算得到。BMP280 的压力输出受温度影响，所以压力补偿必须使用 t_fine。若温度和压力读取相差 100 ms，在温度变化慢的室内影响较小；但在快速温变下，会用不匹配的温度去补偿压力，从而引入温度交叉敏感误差。

## Q11. Madgwick 滤波器 beta 怎么选？快速机动时有什么影响？

### 1. 先说明项目边界

本项目最终展示的是：

- 互补滤波
- Mahony 滤波
- ESKF 静态分析

没有把 Madgwick 滤波作为最终主结果。因此答题时不能硬说“我的 Madgwick 参数是多少”。可以先说明：

> 本项目最终没有把 Madgwick 作为主结果，但从 Madgwick 原理看，beta 的作用是这样的。

### 2. beta 是什么

Madgwick 滤波中，陀螺仪负责预测姿态，加速度计和磁力计负责修正姿态。`beta` 控制修正项强度。

简单理解：

- `beta` 大：更相信加速度计/磁力计修正，收敛快，但动态时容易被干扰。
- `beta` 小：更相信陀螺积分，动态更平滑，但长期漂移更明显。

### 3. 快速急转弯时为什么 beta 要小心

飞机快速急转弯时，加速度计测到的不再只是重力，还包含：

- 离心加速度
- 机动加速度
- 振动

Madgwick 的姿态修正默认加速度计主要测重力方向。如果这个假设不成立，过大的 beta 会把非重力加速度误认为姿态倾斜，从而把姿态拉错。

### 4. 200 deg/s 持续 0.5 秒的影响

角速度 200 deg/s 持续 0.5 s，姿态变化量约：

```text
200 × 0.5 = 100°
```

这是很强的动态机动。此时应降低加速度计修正权重，也就是降低 beta，或者动态调整 beta。

### 5. 可直接背的回答

> 本项目最终展示的是互补滤波和 Mahony 滤波，没有把 Madgwick 作为主结果。如果从 Madgwick 原理看，beta 是梯度下降修正项权重。beta 越大，越相信加速度计和磁力计，收敛快但动态机动时容易被非重力加速度误导；beta 越小，越相信陀螺积分，动态更平滑但长期漂移更大。飞机快速急转时，加速度计测到的不只是重力，所以 beta 应该适当降低或自适应调整。

## Q12. Roll/Pitch 和 Yaw 精度是多少？为什么 Yaw 比 Roll/Pitch 差？

### 1. 本项目可报数据

静态姿态实验：

- ESKF roll 标准差：`0.074643°`
- ESKF pitch 标准差：`0.132308°`
- 静态稳定 RMS：`0.151800°`

航向残差补偿：

- 平均绝对误差：`1.907°`
- 最大绝对误差：`6.940°`

### 2. 为什么 Roll/Pitch 比 Yaw 好

静止时，加速度计可以测到重力方向。

重力方向可以约束：

- Roll
- Pitch

因为板子前后倾、左右倾，重力在三轴上的投影会变化。

但是 Yaw 是绕重力轴旋转。比如板子水平放在桌上，原地转一圈，重力方向在传感器坐标里的投影基本不变。

所以：

```text
加速度计无法观测 Yaw
```

Yaw 必须依赖：

- 磁力计
- GPS 航向
- 视觉
- 双天线 GNSS

而磁力计很容易受到磁环境影响，所以 Yaw 通常比 Roll/Pitch 差。

### 3. 如果去掉磁力计，Yaw 10 分钟会漂多少

去掉磁力计后，Yaw 只能靠陀螺积分。

漂移估算公式：

```text
Yaw 漂移 ≈ 陀螺零偏 × 时间
```

10 分钟：

```text
600 s
```

如果陀螺 Z 轴零偏是 `0.05 deg/s`，则：

```text
0.05 × 600 = 30°
```

如果零偏是 `0.01 deg/s`，则：

```text
0.01 × 600 = 6°
```

所以具体漂移要看实测陀螺零偏，但一定会随时间累积。

### 4. 可直接背的回答

> 本项目静态姿态中，Roll 标准差约 0.0746°，Pitch 标准差约 0.1323°，静态稳定 RMS 约 0.1518°。航向补偿后平均绝对误差约 1.907°，最大约 6.940°。Yaw 比 Roll/Pitch 差，是因为加速度计只能观测重力方向，可以约束 Roll 和 Pitch，但绕重力轴旋转的 Yaw 对加速度计不可观，必须依赖磁力计，而磁力计又容易受环境干扰。去掉磁力计后，Yaw 会按陀螺零偏乘时间累积漂移，比如 0.05 deg/s 的零偏 10 分钟会漂约 30°。

## Q13. 卡尔曼滤波 Q 矩阵代表什么？Q 过小或过大会怎样？

### 1. 老师到底在问什么

老师想确认你理解 KF 调参，而不是只会调用公式。

Kalman 滤波核心是平衡：

- 模型预测
- 传感器观测

### 2. Q 和 R 的区别

`Q` 是过程噪声协方差。

它表示：

> 我对系统模型预测有多不信任。

`R` 是观测噪声协方差。

它表示：

> 我对传感器测量有多不信任。

### 3. Q 过小会怎样

Q 过小表示滤波器过度相信模型预测。

现象：

- 曲线很平滑。
- 对真实变化反应慢。
- 楼梯高度变化时可能跟不上。
- 严重时滤波器过度自信，真实状态变化后仍不修正，可能发散。

这叫：

- over-confidence
- filter divergence

### 4. Q 过大会怎样

Q 过大表示滤波器认为模型很不可靠，于是更相信观测。

现象：

- 曲线跟着测量抖动。
- 滤波效果变差。
- 近似退化成直接测量或加权平均。

### 5. 项目边界

如果老师问“你的高度 KF”，但本项目最终主要展示 BMP280 相对高度实验，而不是完整 GPS/气压高度 KF，可以说：

> 本项目主要完成 BMP280 相对高度验证，KF 参数属于扩展分析。如果使用高度 KF，Q 应根据真实垂直运动变化和气压高度模型误差来设置。

### 6. 可直接背的回答

> Q 是过程噪声协方差，表示模型预测的不确定性；R 是观测噪声协方差，表示传感器测量的不确定性。Q 设得太小，滤波器过度相信模型，输出很平滑但跟踪真实变化慢，严重时过度自信甚至发散；Q 设得太大，滤波器过度相信观测，输出会抖，滤波效果减弱，接近直接跟随测量。

## Q14. GPS/气压互补滤波 alpha=0.98 对应多长时间常数？气压天气漂移怎么办？

### 1. 项目边界

本项目最终没有把 GPS/气压动态互补滤波作为主结果，完成的是：

- BMP280 楼梯相对高度实验。
- GNSS NMEA 解析。
- 静态 GPS 数据分析。

所以如果老师问，可以先说：

> 本项目没有把 GPS/气压动态互补作为最终主结果，但从一阶互补滤波原理看，alpha 可以这样理解。

### 2. alpha 和时间常数关系

一阶滤波中，近似时间常数：

```text
tau = dt / (1 - alpha)
```

如果：

```text
alpha = 0.98
dt = 0.02 s
```

则：

```text
tau = 0.02 / 0.02 = 1 s
```

意思是滤波器大约用 1 秒时间响应变化。

### 3. 台风前气压下降 3 hPa 会造成多大高度误差

近似关系：

```text
1 hPa ≈ 8.4 m
```

所以：

```text
3 hPa ≈ 25.2 m
```

如果不更新 `P0`，气压高度会慢慢漂移约 25 m。

### 4. 怎么补偿

方法：

1. 定期用 GPS 高度或已知高度修正 `P0`。
2. 引入气象站参考气压。
3. 对气压慢漂建模，把 `P0` 作为状态估计。
4. 动态调整 alpha，在长期慢漂时增加 GPS/外部参考权重。

### 5. 可直接背的回答

> 本项目没有把 GPS/气压动态互补作为最终主结果。如果按 alpha=0.98、dt=0.02s 计算，时间常数约为 1s。气压高度对天气很敏感，1hPa 约等于 8.4m 高度变化，所以 1 小时下降 3hPa 会造成约 25m 的高度漂移。解决方法是定期用 GPS 或已知高度重置 P0，或者把 P0 漂移作为状态估计，而不能长期只依赖气压计。

## Q15. I2C 总线上三个传感器如何调度？中断竞争怎么处理？

### 1. 老师到底在问什么

老师想看你是否理解嵌入式实时系统中总线共享和中断竞争。

本项目 I2C 总线挂了：

- BMP280：`0x76`

### 2. I2C 总线基本特点

I2C 是主从总线。

ESP32-S3 是主设备，传感器是从设备。所有读写都由 ESP32-S3 发起。

关键点：

> 同一时刻 I2C 总线上只能有一个事务。

所以不能同时读 MPU、HMC 和 BMP，必须排队。

### 3. 如何调度读取顺序

按传感器更新需求：

- IMU 频率最高，优先读。
- 磁力计频率中等。
- BMP280 气压变化慢，频率可以最低。

可用调度方式：

```text
每个循环读 IMU
每 N 个循环读磁力计
每 M 个循环读 BMP280
```

这样既保证姿态融合实时性，也减少总线占用。

### 4. MPU 中断在 BMP 读取途中触发会怎样

如果 MPU 的 data ready 中断在 BMP280 I2C 读取途中触发：

- 硬件上不会打断当前 I2C 事务。
- 中断只是通知“MPU 数据好了”。
- 正确做法是在中断服务函数里只置标志位。
- 当前 BMP 读完后，主循环或任务再处理 MPU 读取。

### 5. 多任务竞争怎么处理

如果 FreeRTOS 多个任务都可能访问 I2C，需要 mutex：

```text
take mutex
I2C read/write
give mutex
```

否则任务切换可能导致两个任务同时操作 I2C 驱动，造成通信异常。

### 6. 可直接背的回答

> I2C 是主从总线，ESP32-S3 控制所有时序，所以三个传感器不能同时读，必须由主控调度。我会让 IMU 高频读取，磁力计和 BMP280 低频读取。MPU 中断在 BMP280 读取途中触发时，不会打断当前 I2C 事务，中断里只置 data-ready 标志，等当前事务结束后再读 MPU。如果多个 FreeRTOS 任务访问 I2C，则必须用 mutex 保护总线。

## Q16. MPU6050 安装倾斜能被六位置标定补偿吗？

### 1. 老师到底在问什么

老师想确认你理解“标定参数”和“机械安装状态”绑定。

如果传感器焊死在 PCB 上，安装误差是固定的，可以通过标定补偿。

如果传感器用杜邦线晃来晃去，安装误差每次都变，标定参数就失效。

### 2. 安装倾斜是什么

安装倾斜指的是：

> 传感器芯片坐标系和 PCB 板坐标系不完全重合。

比如板子 X 方向和 MPU 芯片 X 方向差了 1°。

这会导致真实 X 方向加速度投影到 Y/Z 轴，看起来像轴间耦合。

### 3. 六位置标定能不能补偿

能，但有前提：

- 传感器固定在 PCB 上。
- 标定后相对位置不再变化。
- 后续使用时仍保持同一安装状态。

补偿发生在：

```text
a_cal = A_inv · (a_raw - c)
```

其中 `A_inv` 的非对角项吸收安装角误差。

### 4. 什么情况不能补偿

如果传感器不是固定的，比如：

- 杜邦线连接
- 模块悬空
- 每次插拔角度不同

那安装关系会变。一次标定得到的 `A_inv` 只对当时状态有效，换个姿态就不准确。

### 5. 可直接背的回答

> 如果 MPU6050 固定焊接在 PCB 上，安装倾斜是固定误差，可以被六位置 12 参数标定补偿，主要体现在 A_inv 的非对角项里。补偿发生在 `A_inv · (raw-c)` 这一步。但如果传感器用杜邦线连接，每次相对角度都变，那么安装误差不是固定的，之前的标定参数就不能长期有效。

## Q17. 无人机电机启动后磁力计跳变，有哪些解决方案？

### 1. 老师到底在问什么

老师想看你能不能从系统工程角度解决磁干扰问题，而不是只说“屏蔽一下”。

电机启动后，磁力计跳变可能来自：

- 电机磁场。
- 电调电流。
- 电源线大电流。
- 机架铁磁材料。
- 高频 EMI。

### 2. 方案一：磁力计远离干扰源

做法：

- 用延伸杆把磁力计远离电机、电调和大电流线。

优点：

- 从源头降低干扰。
- 简单直接。

缺点：

- 结构复杂。
- 线缆变长，可能引入通信和振动问题。

### 3. 方案二：电流补偿

做法：

- 测量电机电流。
- 建立磁干扰和电流之间的模型。
- 飞行中根据电流补偿磁力计。

类似 ArduPilot 的 COMPASS_MOT 思路。

优点：

- 能补偿随油门变化的磁干扰。

缺点：

- 需要电流传感器。
- 需要专门标定。
- 对复杂干扰不一定完全有效。

### 4. 方案三：降低磁力计权重

做法：

- 当检测到磁场模长异常时，降低磁力计在姿态融合里的权重。
- 短时间更多依赖陀螺和其他传感器。

优点：

- 软件实现，不需要改硬件。

缺点：

- 长时间不用磁力计，Yaw 会漂。

### 5. 方案四：双天线 GNSS 测向

做法：

- 用双 GNSS 天线直接测航向。

优点：

- 不受电机磁场影响。
- 大型无人机上更可靠。

缺点：

- 成本高。
- 需要足够天线基线。
- 室内或遮挡环境不好用。

### 6. 我会选什么

优先级：

1. 先做硬件隔离：磁力计远离电机和大电流线。
2. 再做磁异常检测和电流补偿。
3. 如果是大型平台，再考虑双天线 GNSS。

### 7. 可直接背的回答

> 我会先从硬件上把磁力计远离电机、电调和大电流线，这是最直接有效的；其次可以做类似 COMPASS_MOT 的电流补偿，根据电流估计磁干扰；软件上还可以检测磁场模长异常，在异常时降低磁力计权重；如果是大型无人机，可以用双天线 GNSS 测向替代磁罗盘。单纯屏蔽不一定可靠，因为低频磁场很难用普通屏蔽完全消除。

## Q18. 如果重新设计 PCB，会改哪些 Layout？

### 1. 老师到底在问什么

老师想看你是否能批判性分析自己的硬件，而不是只说“下次优化布局”。

### 2. 不知道铺铜覆盖率怎么办

如果你没有精确计算背面铺铜覆盖率，不要乱报数字。

可以说：

> 我没有在答辩前精确统计铺铜覆盖率，但从这次调试结果看，下一版我会重点改磁力计位置、电源去耦和 I2C 参考地。

### 3. 改进点一：磁力计远离干扰源

问题：

- HMC5883L 对磁环境很敏感。
- 靠近 ESP32、USB、电源走线或大电流路径会影响 yaw。

改进：

- 把磁力计放到 PCB 边缘。
- 远离天线、电源芯片、USB 口、大电流走线。

收益：

- 降低磁场均值偏移和模长波动。
- 有助于降低航向角误差。

### 4. 改进点二：去耦电容靠近传感器

问题：

- 电源噪声会影响传感器读数。

改进：

- 每个传感器 VCC 附近放 0.1 uF 去耦电容。
- BMP280、HMC5883L、MPU 电源旁边尽量短回路。

收益：

- 降低电源纹波。
- 减少 uT 级磁场波动和 I2C 通信异常概率。

### 5. 改进点三：I2C 走线和参考地

问题：

- I2C 是数字信号，如果走线长、参考地不连续，容易产生毛刺。

改进：

- SDA/SCL 尽量短。
- 走线下方保持连续地平面。
- 上拉电阻靠近总线合适位置。
- 避免穿过电源噪声区域。

收益：

- 提高 400 kHz I2C 稳定性。
- 降低通信错误和边沿串扰。

### 6. 改进点四：地平面连续

问题：

- 地平面被切割会导致回流路径变长，增加 EMI。

改进：

- 保持连续铺地。
- 高速数字区和敏感传感器区合理分区。

收益：

- 减少数字噪声耦合到模拟/磁传感器。

### 7. 可直接背的回答

> 我没有精确统计铺铜覆盖率，但如果重新设计，我会优先改三点。第一，把 HMC5883L 放到远离 ESP32 天线、USB 和电源走线的位置，降低磁干扰。第二，把各传感器去耦电容更靠近电源引脚，减小电源纹波。第三，优化 I2C 走线，保证 SDA/SCL 短且有连续参考地，避免跨分割地。这样可以提高 I2C 稳定性，并降低磁力计模长波动和航向误差。

## Q19. 系统最终精度是多少？误差预算怎么做？

### 1. 老师到底在问什么

这是综合题。老师想看你是否知道每个结果的瓶颈在哪里，而不是只报一个数字。

### 2. 本项目可报精度

| 指标 | 当前结果 | 主要瓶颈 |
|---|---:|---|
| Roll 静态标准差 | 0.0746° | 加速度计噪声、安装误差、桌面水平度 |
| Pitch 静态标准差 | 0.1323° | 加速度计噪声、安装误差、桌面水平度 |
| 姿态静态 RMS | 0.1518° | 静态数据质量、滤波参数 |

| GPS 静态最大漂移 | 0.171 m | 卫星几何、室外环境、接收机噪声 |

### 3. Roll/Pitch 的瓶颈

Roll/Pitch 主要由重力方向约束，因此瓶颈是：

- 加速度计标定残差。
- 板子放置水平度。
- 静态振动和噪声。
- 滤波参数。

纯软件提升空间：

- 继续优化加速度计标定。
- 使用更好的异常检测和滤波。
- 提升有限，因为硬件和摆放误差已经成为重要因素。

### 4. Yaw/航向的瓶颈

Yaw 的瓶颈主要是磁力计：

- 磁环境干扰。
- HMC5883L 安装位置。
- 椭球拟合数据覆盖程度。
- 参考方向摆放误差。

纯软件提升空间：

- 鲁棒椭球拟合。
- 磁异常检测。
- 航向残差补偿。
- 但如果硬件环境磁干扰很大，纯软件提升有限。

### 5. 高度的瓶颈

BMP280 相对高度短时间表现较好，但长期高度瓶颈不是分辨率，而是：

- `P0` 参考气压。
- 天气气压漂移。
- 温度交叉敏感。
- 气流扰动。

### 6. 可直接背的回答

> 当前 Roll/Pitch 静态标准差分别约 0.0746° 和 0.1323°，静态 RMS 约 0.1518°；航向补偿后平均绝对误差约 1.907°，最大约 6.940°；BMP280 楼梯相对高度误差约 0.030 m；GPS 静态最大漂移约 0.171 m。误差瓶颈上，Roll/Pitch 主要受加速度计标定残差和平台水平度影响，Yaw 主要受磁干扰和参考方向影响，高度主要受 P0 和气压环境影响，而不是 BMP280 分辨率。现有硬件上，软件还能通过鲁棒磁标定、磁异常检测和更好的滤波继续改善，但如果磁环境和安装位置不变，Yaw 提升会有限。

# 最后提醒：哪些话不能硬说

以下说法答辩时不要讲：

- 不要说动态 GPS 轨迹已经完整完成。
- 不要说航向所有方向都达到 ±2°。
- 不要说陀螺仪零偏漂移已经严格下降 10×。
- 不要说 AI 模型已经部署到 ESP32-S3 板端。
- 不要说 GNSS/INS 达到厘米级动态定位。
- 不要说 BMP280 绝对高度很准，应强调短时间相对高度。
- 不要说 Madgwick 是你的主结果，因为项目主展示是互补滤波和 Mahony。

答辩时更稳的说法是：

> 本项目已经完成硬件采集、传感器驱动、加速度计与磁力计标定、姿态融合、AI 离线去噪、BMP 相对高度和 GNSS 静态解析。部分动态 GPS/INS 和板端 AI 部署属于后续扩展，没有作为最终主结果。
